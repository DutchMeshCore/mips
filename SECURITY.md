# Security Policy

If you identify a vulnerability that could impact mesh safety, privacy, infrastructure, or legal compliance, do not open a public issue first.

## Report a vulnerability

Open a private security advisory in GitHub or contact maintainers directly through agreed private channels.

Include:

- Clear description of the issue
- Impact and affected components
- Reproduction details
- Suggested mitigation if known

## Disclosure approach

- Maintainers acknowledge reports as soon as possible.
- Sensitive details stay private until mitigation is available.
- Public postmortems should focus on lessons and operational improvements.
